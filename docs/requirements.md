# 需求文档 - 订单转换助手

**版本:** 1.0  
**日期:** 2026-03-03  
**负责人:** Roadmap (PM)

---

## 1. 背景

### 1.1 现状
销售人员每天需要：
1. 登录多个客户的外部订单系统（BYD、华为、美团等）
2. 手动查询与自己公司相关的订单
3. 把外部订单信息转成公司内部订单格式
4. 提交到内部 ERP 系统

### 1.2 痛点
- **重复劳动** — 每天都要做相同的操作
- **容易出错** — 手动转换容易遗漏或填错
- **效率低** — 每个订单耗时 10-15 分钟
- **无法追踪** — 没有历史记录和状态管理

---

## 2. 目标用户

- 公司销售团队
- 需要处理多客户订单的商务人员
- **用户规模:** 现阶段单机本地使用
- **订单量:** 每天约 4-5 单

---

## 3. 核心功能

### 3.1 MVP 功能（P0）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **外部订单抓取** | 对接客户订单系统，自动拉取订单数据 | P0 |
| **订单映射转换** | 外部订单字段 → 内部订单格式自动转换 | P0 |
| **批量处理** | 支持一次处理多个订单 | P0 |
| **人工审核** | 转换后支持人工确认/修改再提交 | P0 |
| **提交内部系统** | 一键提交到公司内部订单系统 | P0 |

### 3.2 增强功能（P1）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **订单状态同步** | 跟踪已提交订单的状态 | P1 |
| **多客户配置** | 支持不同客户的不同订单系统 | P1 |
| **智能映射学习** | 记住映射规则，自动匹配 | P1 |
| **异常处理** | 价格差异、SKU 不匹配等异常提醒 | P1 |

---

## 4. 技术约束

| 项目 | 详情 | 影响 |
|------|------|------|
| **外部系统** | 网页爬虫 + 登录 + 短信验证码 | 需要浏览器自动化 + 人工介入验证码 |
| **内部系统** | 网页表单提交 | 需要浏览器自动化 |
| **字段映射** | 无现成对应关系 | 需要设计映射配置界面 |
| **使用规模** | 单机本地使用 | 轻量级应用，不需要高并发 |
| **安全要求** | 账号密码 + 短信验证 | 本地加密存储凭证 |

---

## 5. 非功能需求

### 5.1 性能
- 单次订单抓取 < 30 秒
- 映射转换 < 5 秒
- 界面响应 < 1 秒

### 5.2 安全
- 凭证本地加密存储
- 不上传任何敏感数据到云端
- 支持密码保护应用

### 5.3 可用性
- 支持断点续传
- 操作历史可追溯
- 错误提示清晰

### 5.4 可维护性
- 爬虫配置可热更新
- 支持新增客户系统无需改代码
- 完整的日志记录

---

## 6. 用户故事

### 6.1 日常使用流程

```
1. 打开应用 → 选择客户系统
2. 输入验证码（如需）→ 自动登录
3. 点击"同步订单" → 抓取新订单
4. 查看映射结果 → 人工确认/修正
5. 点击"提交 ERP" → 自动填写并提交
6. 查看提交状态 → 完成
```

### 6.2 配置流程

```
1. 新增客户系统配置
2. 定义字段映射规则
3. 定义物料/SKU 映射
4. 测试连接和抓取
5. 保存配置
```

---

## 7. 验收标准

### 7.1 MVP 验收
- [ ] 能成功登录至少 1 个客户系统
- [ ] 能抓取订单数据
- [ ] 能完成字段映射转换
- [ ] 能提交到内部系统
- [ ] 用户能独立完成全流程

### 7.2 质量要求
- 无严重 Bug
- 关键流程有错误处理
- 有基本的日志记录

---

## 8. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 短信验证码需人工介入 | 无法完全自动化 | 设计友好的人工输入界面 |
| 网页结构变化 | 爬虫失效 | 可配置的 selector，快速更新 |
| 账号安全 | 凭证泄露 | 本地加密存储，不上传云端 |
| 反爬虫机制 | 被封 IP | 控制频率，支持代理 |
