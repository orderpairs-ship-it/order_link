## ⏰ 每日定时任务

### 09:00 - 晨会同步
**负责人:** Roadmap
- 读取 STATE.yaml，了解各任务状态
- 检查是否有阻塞项
- 在群里发布晨会摘要：
  - 昨日完成
  - 今日计划
  - 需要关注的风险

### 12:00 - 进度检查
**负责人:** Roadmap
- 查询各成员进度（如未主动更新）
- 更新 STATE.yaml
- 如有阻塞，协调解决

### 18:00 - 日报总结
**负责人:** Roadmap
- 汇总今日完成情况
- 更新 STATE.yaml 进度
- 在群里发布日报：
  - 今日完成
  - 未完成项及原因
  - 明日计划
  - 需要 Boss 知晓的事项

## 🔄 持续任务（事件驱动）

| 触发条件 | 任务 | 负责人 |
|----------|------|--------|
| PR 创建 | Code Review (24h 内) | Backend / Canvas |
| Bug 报告 | 调查 + 修复 | 相关成员 |
| Bug 修复 | 验证 + 关闭 | Quality |
| 代码推送 | CI/CD 监控 | DevOps |
| 任务完成 | 更新 STATE.yaml | 任务负责人 |
| 遇到阻塞 | 更新 blocked + @Roadmap | 任务负责人 |

## 📅 每周任务

### 周一 09:00 - 周计划
**负责人:** Roadmap
- 回顾上周进度
- 更新 GOALS.md（本周 OKR）
- 创建/更新本周任务
- 在群里发布周计划

### 周五 18:00 - 周复盘
**负责人:** Roadmap
- 汇总本周完成情况
- 记录关键决策到 DECISIONS.md
- 更新 PROJECT_TRACKING.md
- 在群里发布周报

## 📝 执行说明

**Roadmap 自主执行:**
- 每日 09:00 / 12:00 / 18:00 自动同步
- 无需 Boss 提醒
- 如有异常主动报告

**团队成员期望:**
- 完成任务后立即更新 STATE.yaml
- 遇到阻塞立即 @Roadmap
- 每日至少同步一次进度（可在 18:00 前）
