# 设计对比分析 - M1 线框图 vs OrderLink 高保真参考

**日期:** 2026-03-03  
**目的:** 对比 Boss 提供的 OrderLink 设计参考，评估 M2 迭代方向

---

## 布局对比

| 维度 | M1 线框图 (当前) | OrderLink 参考 | 差异分析 |
|------|-----------------|---------------|---------|
| **主布局** | 左侧快捷操作 + 右侧内容区 | 双栏并排 (左列表 + 右映射) | OrderLink 更聚焦单订单处理 |
| **导航** | 顶部 Header + 配置页独立 | 侧边导航栏 (5 个模块) | OrderLink 导航更清晰 |
| **客户切换** | 表格中展示多客户 | 顶部下拉选择器 | OrderLink 专注单客户工作流 |
| **状态展示** | 今日概览卡片 (3 个指标) | 状态仪表 (Pending/Review/Completed) | 逻辑一致，文案不同 |

---

## 核心差异

### 1. 工作流模式

**M1 设计:** 多客户订单列表 → 批量处理
```
适合场景：运营需要同时处理多个客户的订单
```

**OrderLink:** 单客户专注模式 → 深度处理
```
适合场景：运营逐个客户处理，需要对比外部/内部数据
```

### 2. 映射界面

**M1 设计:** 弹窗形式
- 字段映射弹窗
- 物料映射弹窗
- 配置完成后返回主界面

**OrderLink:** 双栏并排
- 左侧：外部订单原始数据
- 右侧：内部映射结果实时预览
- 优势：可视化对比，减少认知负担

### 3. 状态体系

| M1 状态 | OrderLink 状态 | 映射建议 |
|--------|---------------|---------|
| 🟡 待映射 | Pending | 一致 |
| 🟢 已完成 | Completed Today | 一致 |
| 🔴 失败 | Requires Review | OrderLink 更温和 |
| - | 🟠 Price Discrepancy | M1 缺少价格异常状态 |
| - | 🔴 SKU Unrecognized | M1 用 ⚠ 未匹配表示 |

### 4. 导航结构

**M1:**
```
主界面 → 配置页 (独立页面)
```

**OrderLink:**
```
侧边导航:
- Workbench (主工作台)
- Mapping Rules (映射规则)
- Automation Rules (自动化规则)
- History (历史记录)
```

**分析:** OrderLink 的导航更模块化，适合复杂配置场景

## 决策建议

### M2 高保真设计方向

**推荐:** 融合方案 - 结合 M1 的完整性 + OrderLink 的可视化优势

**核心理念:**
1. **双视图模式:** 列表视图 (批量) + 双栏视图 (深度处理)
2. **渐进式披露:** 简单场景用列表，复杂场景用双栏
3. **状态细化:** 补充价格异常、SKU 未识别等细分状态

---

**结论:** M1 线框图功能完整，OrderLink 在可视化对比上更优。M2 建议融合两者优势。
